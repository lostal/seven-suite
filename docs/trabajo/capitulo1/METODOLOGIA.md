Se emplea el Proceso Unificado (RUP) adaptado a desarrollo individual, organizado en cuatro iteraciones alineadas con capítulos TFG.

- Iteración 1 — Requisitos (Cap. 2)<br>
  Sesiones GRUPOSIETE para capturar necesidades, modelo dominio, glosario, actores, casos uso detallados/priorizados, prototipos UI.
- Iteración 2 — Análisis y diseño (Cap. 3)<br>
  Clases análisis → diseño, diagramas UML (casos uso, clases, paquetes, despliegue), arquitectura (Next.js/Supabase/M365), esquema PostgreSQL.
- Iteración 3 — Implementación (Cap. 4)<br>
  MVP incremental: auth, reservas, admin panel. Pruebas unitarias/E2E (Vitest/Playwright). Despliegue Vercel con CI/CD GitHub Actions.
- Iteración 4 — Evaluación (Cap. 5)<br>
  Métricas cobertura/usabilidad, validación requisitos, análisis decisiones técnicas, limitaciones, roadmap expansión (visitas, mapa 3D).

Soporte: GitHub (trazabilidad commits), Notion (Kanban tareas), Draw.io/Lucidchart (UML). Feedback iterativo semanal.
